# 规则能力矩阵

本文是 `hongxunpan/validator` 的规则能力真相源，用于说明每条规则当前是否已支持、是否进入规划、是否应由 adapter 承接，以及不受理原因。

README 只保留入口说明；规则能力、状态和规划批次以本文及对应版本 tag 为准。

速查时可直接阅读“原因 / 说明”列，它就是每条规则的一句话介绍；README 只保留分组概览，避免重复维护真相源。

---

## 1. 状态标记

| 标记 | 状态 | 含义 |
| --- | --- | --- |
| ✅ | 已支持 | 当前 core 已实现，并应有测试覆盖 |
| 🟡 | 计划中 | 已进入规则补齐路线，默认准备补到 core |
| 🔵 | Adapter | 需求合理，但不属于低依赖 core，应由框架 / 项目 adapter 承接 |
| ⛔ | 不受理 | 与本包边界冲突，未来默认也不接 |
| ⚪ | 待评估 | 暂未承诺，等待真实场景或 issue 推动 |

## 2. 字段说明

| 字段 | 说明 |
| --- | --- |
| 状态 | 使用 emoji + 文本双标，便于快速扫描和文本搜索 |

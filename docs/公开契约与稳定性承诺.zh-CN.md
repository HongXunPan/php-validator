# 公开契约与稳定性承诺

本文档用于说明 `hongxunpan/validator` 当前阶段哪些内容属于**稳定公开契约**，哪些内容只属于**内部实现**。

当前仓库仍处于 **pre-1.0** 阶段，但从本文件开始，已文档化的公开能力会尽量保持稳定；若后续必须调整，将在 README / CHANGELOG / 迁移说明中显式说明。

---

## 1. 总原则

### 1.1 输入与输出边界

本包当前的稳定公开结果边界是：

- `validate(...)`
- `validateAndNormalize(...)`
- `validateListAndNormalize(...)`

统一返回：

- `HongXunPan\Validator\Result\ValidationResult`

本包不会为了兼容旧项目，把 core 回退成“数组优先”心智；若业务项目仍需要旧 envelope 或 `*OrThrow -> array`，应在项目适配层自行接回。

### 1.2 Internal 不承诺稳定

以下路径下的所有类，默认都属于**内部运行时实现**：

- `HongXunPan\Validator\Internal\*`

这些类可能在后续版本中：

- 重命名
- 移动目录
- 合并/拆分
- 修改构造器
- 调整内部协作方式

即使它们目前是 `public class`，也**不构成稳定扩展承诺**。

### 1.3 文档化公开面优先

并非所有 `public` 可见的方法都自动构成稳定契约。

只有本文件明确列出的内容，才视为当前阶段的公开稳定面；未列出的 `public` 方法，如果只是为了内部装配、调试或历史过渡而保留，不应被外部项目依赖。

---

## 2. 当前稳定公开契约

## 2.1 公开入口

以下类属于当前稳定公开入口：

- `HongXunPan\Validator\Validator`
- `HongXunPan\Validator\ValidationKernel`

### `Validator` 的稳定职责

- 作为使用侧定义子类的主入口；
- 承接：
  - `validate(...)`
  - `validateAndNormalize(...)`
  - `validateListAndNormalize(...)`
  - `writeValidatedDataTo(...)`
  - `validateAndWriteTo(...)`
- 承接三组扩展数组：
  - `extraRules`
  - `ruleAliases`
  - `ruleMessages`

### `ValidationKernel` 的稳定职责

- 作为更底层的对象级入口；
- 承接：
  - `create(...)`
  - `validate(...)`
  - `validateAndNormalize(...)`
  - `validateListAndNormalize(...)`
  - `writeValidatedDataTo(...)`
  - `validateAndWriteTo(...)`

### 明确不属于稳定承诺的入口泄漏

以下内容虽然当前可见，但**不属于稳定公开契约**：

- `Validator::coreRules()`
- `Validator::extraRules()`
- `Validator::ruleAliases()`
- `Validator::ruleMessages()`
- `Rule\CoreRules`

这些能力当前主要是为了内核装配、测试或历史过渡保留，后续可能继续调整。

---

## 2.2 规则扩展契约

以下类型属于当前稳定扩展面：

- `HongXunPan\Validator\Rule\RuleInterface`
- `HongXunPan\Validator\Rule\AbstractRule`
- `HongXunPan\Validator\Rule\AbstractPresenceRule`
- `HongXunPan\Validator\Rule\AbstractValueRule`
- `HongXunPan\Validator\Rule\PresenceRuleInterface`
- `HongXunPan\Validator\Rule\ValueRuleInterface`
- `HongXunPan\Validator\Rule\ConditionalPresenceRuleInterface`
- `HongXunPan\Validator\Rule\ValueMaterializationRuleInterface`
- `HongXunPan\Validator\Rule\DependentValueRuleInterface`

### 当前稳定的规则扩展方式

使用侧默认应通过：

1. 编写自定义 Rule 类；
2. 在 `Validator` 子类中声明：
   - `extraRules`
   - `ruleAliases`
   - `ruleMessages`

来扩展行为。

### 当前稳定的 Rule 契约语义

对外承诺稳定的，是以下语义：

- `KEY`
- `key()`
- `of(...)`
- `defaultMessage()`
- `displayRuleValue(...)`
- `validate($context)`

以及：

- Presence / Value 分阶段标记；
- `ConditionalPresenceRuleInterface` 表示条件型 presence；
- `ValueMaterializationRuleInterface` 表示会参与值物化；
- `DependentValueRuleInterface` 表示依赖其他 target 可读值的 value 规则。

### 内部不承诺稳定的部分

以下内容即使与规则执行有关，也不属于公开扩展承诺：

- 规则在内核内被分几段调度；
- `Internal\Execution\*`
- `Internal\Rules\*`
- `Internal\Runner\*`

规则实现者不应依赖这些内部调度细节。

---

## 2.3 规则上下文与结果对象

以下对象属于当前稳定公开面：

- `HongXunPan\Validator\Context\RuleContext`
- `HongXunPan\Validator\Context\RuleValueReaderInterface`
- `HongXunPan\Validator\Context\ValidationOptions`
- `HongXunPan\Validator\Context\PathValue`
- `HongXunPan\Validator\Context\PathLabelMap`
- `HongXunPan\Validator\Result\RuleResult`
- `HongXunPan\Validator\Result\ValidationResult`

### `RuleContext` 的稳定职责

- 向 Rule 暴露当前 target 的最小只读上下文；
- 暴露：
  - 当前字段路径
  - 参数显示名
  - 当前值 / 原始值
  - rule 参数
  - 字面量解析
  - 跨字段读取能力
- 其公共构造与公共类型提示不再泄漏 `Internal/*` 运行时类型；

### `RuleResult` 的稳定语义

当前公开稳定的，是其“单规则执行结果”语义：

- `pass(...)`
- `fail(...)`
- `passAndBreak(...)`
- `passed() / failed()`
- `value()`
- `shouldBreak()`
- `exists()`

### `ValidationResult` 的稳定语义

当前公开稳定的，是其“统一对外结果”语义：

- `count()`
- `errors()`
- `detail()`
- `validatedData()`
- `isPassed() / isFailed()`
- `toArray()`

---

## 2.4 输出辅助能力

以下类型属于当前稳定公开能力：

- `HongXunPan\Validator\Output\ValidatedDataWriterInterface`
- `HongXunPan\Validator\Output\ArrayAccessValidatedDataWriter`

当前稳定承诺是：

- v1 只承接“把 `validated_data` 写入已存在目标容器”的能力；
- 目标容器默认支持 `ArrayAccess`；
- core 不负责 DTO 实例化、反射映射、构造器注入或 hydrator 体系。

---

## 2.5 异常类型

以下异常类型属于当前公开面，可被外部项目捕获：

- `HongXunPan\Validator\Exception\ExceptionInterface`
- `HongXunPan\Validator\Exception\ValidatorException`
- `HongXunPan\Validator\Exception\InvalidRuleClassException`
- `HongXunPan\Validator\Exception\RuleNameReservedException`
- `HongXunPan\Validator\Exception\InvalidValidatedDataTargetException`

但需要注意：

- **异常类型**属于公开面；
- **异常 message 文本**不承诺逐字稳定。

---

## 2.6 core 规则类

当前内建在 `Rule\Presence / Rule\Type / Rule\Transform / Rule\Assert / Rule\Collection` 下的 canonical core 规则类，默认属于公开使用面的一部分。

对外稳定承诺的是：

- 这些规则类可被使用侧直接引用；
- 其 `KEY / key() / of(...) / defaultMessage() / validate(...)` 语义与 DSL key 对应关系稳定；
- alias 查找顺序固定为：
  1. 先查真实规则名；
  2. 真实规则不存在时再查 alias；
  3. alias 命中后执行最终规则类。

不承诺稳定的是：

- `CoreRules` 这种 core rule 装配器本身；
- 规则在 internal 中如何缓存、如何被 runner 调度的细节。

---

## 3. pre-1.0 变更承诺

当前仓库仍处于 `0.x` 阶段，因此并不承诺“像 1.0 以后那样极端保守”。

但从本文件开始，项目承诺：

1. **不会随意破坏已文档化的公开契约**；
2. 若必须调整已文档化公开面，会至少同步：
   - README
   - CHANGELOG
   - 必要时补迁移说明
3. `Internal/*` 的变化不视为 breaking public API；
4. 单个项目历史 helper 的兼容需求，不会直接反向塑造 core 默认契约。

---

## 4. 对接项目时的推荐理解

### 推荐依赖

项目接入时，建议只依赖：

- `Validator` 子类扩展方式；
- `ValidationResult` 结果对象；
- `RuleInterface` 规则扩展契约；
- README / docs 中已文档化的 DSL 与行为说明。

### 不推荐依赖

项目接入时，不建议依赖：

- `Internal/*`
- `Rule\CoreRules`
- internal runner / state / output / context 目录结构
- 任何未在 README / docs 中声明为公开契约的辅助方法
